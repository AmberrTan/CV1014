## Design Context

### Users
This project is a class project intended for lecturers, classmates, and project evaluators reviewing a gym recommendation system. The interface should help them quickly understand the product, explore gym options, and see that the system can search, compare, and recommend gyms clearly and confidently during a demo.

### Brand Personality
The brand personality is Youth, Workout, Gym. The interface should feel polished, confident, and presentation-ready rather than casual or playful. It should communicate that the system is practical, well-organized, and trustworthy for comparing fitness options.

### Aesthetic Direction
Use a light-mode-only interface. Lean into a premium, refined presentation with strong structure, clear hierarchy, warm surfaces, and controlled accent usage. The current warm neutral palette and editorial feel are acceptable as long as the result feels more intentional than generic. Avoid anything that looks noisy, gimmicky, childish, or obviously AI-generated.

### Design Principles
1. Design for demo clarity first: every screen should help evaluators understand the system's core value within a few seconds.
2. Keep the tone premium and confident: strong hierarchy, disciplined spacing, and deliberate accents over decorative clutter.
3. Make comparison and recommendation flows feel trustworthy: prioritize legibility, scannability, and clear reasoning over flashy effects.
4. Stay light-mode and presentation-safe: avoid risky styling choices that reduce readability or look unstable in a classroom setting.
5. Support the class-project context: the UI should showcase system capability clearly, not imitate a full commercial fitness brand experience.
